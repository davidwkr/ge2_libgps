# Implementation Plan - Movistar GNSS Fix (WiFi Optimized)

The device is currently identified as **AR PERSONAL** (alpha) but reports MNC **72236** (numeric). This discrepancy, combined with an LTE-only SUPL configuration and proprietary CMCC mode, is preventing a GNSS fix. Boot logs show stale ephemeris and assist flag errors.

I will apply the **Universal Master Fix**, which includes specific Control Plane support required for Personal Argentina.

## User Review Required

> [!IMPORTANT]
> This fix involves modifying `/vendor/etc`, `/system/etc`, and `/data/gnss` partitions. It requires the device to be rooted and have its partitions remountable as RW.
> A **REBOOT** is mandatory after application to reload the GNSS engine.

## Proposed Changes

### GNSS Configuration (Personal/Universal)
I will apply the [final_super_universal_fix.sh](file:///home/latte/Documents/repos/alt/ge2_libgps/final_super_universal_fix.sh) logic:

1.  **SUPL Configuration**: Update `/vendor/etc/supl.xml` and `/data/gnss/supl/supl.xml` with:
    -   `INTERFACE="any"` (WiFi-friendly).
    -   Dedicated **Control Plane** block for Personal.
    -   Google SUPL IP (`172.217.192.192`) to bypass DNS issues.
2.  **Engine Configuration**: 
    -   Set `CMCC-ENABLE="FALSE"` (Switch to Standard A-GNSS).
    -   Inject `CP-ENABLE="TRUE"` in `/vendor/etc/config.xml`.
3.  **Framework Sync**: Update `gps.conf` in `/vendor/etc` and `/system/etc` to match the engine capabilities.
4.  **State Clear**: Deep clean of `/data/gnss/` state.

## Verification Plan

1.  **Audit Check**: Run [./check_device_config.sh](file:///home/latte/Documents/repos/alt/ge2_libgps/check_device_config.sh) after applying the fix to confirm:
