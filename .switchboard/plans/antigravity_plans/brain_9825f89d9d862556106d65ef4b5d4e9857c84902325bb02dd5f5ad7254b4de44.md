# Implementation Plan - Map UI Stability

The goal is to eliminate re-composition loops in the `MapViewerScreen` and stabilize the track rendering to prevent unintended color shifts.

## Proposed Changes

### [Component Name]
#### [MODIFY] [MapViewerScreen.kt](file:///home/latte/Documents/repos/alt/STViewer/app/src/main/java/com/iodvd/stviewer/ui/MapViewerScreen.kt)
- Introduce a `mapZoom` state updated by an `OnScrollListener` to provide a stable, Compose-friendly zoom value.
- Move `displayedMarkerCount` calculation to a `remember(visibleRecords, mapZoom, appliedDrawMode, isPinVisible)` block.
- Refactor `decimatedRecords` to use the stabilized `mapZoom`.
- Move the `zoomToBoundingBox` logic to a `LaunchedEffect` that observes the data/filter signature.
- Remove all `state = newValue` calls from the `AndroidView.update` block.
